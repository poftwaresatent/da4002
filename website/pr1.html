ITADS Project 1
sorting algorithm benchmarks

Complexity analysis is an important tool to evaluate possible solutions for a given problem. Lecture 4 and the course book by Loudon discuss several theoretical aspects thereof. The first project complements the theoretical treatment with hands-on experience. It provides a practical and empirical understanding of the theory, and prepares the participants to make informed choices in real-life programming situations.

The deadline for handing in the source code and the report is Friday, October 4, 2013, at 18h00. During the three lab sessions on Sep 27, Oct 1, and Oct 4, the students can get help from the lecturer and assistant in the Unix room B231c. We will help with any question that may arise, except of course implement the project itself, without any influence on the final evaluation. So please do take advantage of these valuable hours to advance on your project.

You are provided with the following files as a starting point. Read them carefully, as they contain a lot of very relevant and detailed information.

  • Project 1 Assignment in PDF format. Follow the given instructions very carefully.
  • Project 1 Report Template in PDF format. This is an example report with detailed instructions and explanations about how to write a good report. The report counts for approximately half the points of the project, so make sure that it receives the attention that it deserves.
  • The source code insertion-sort-benchmark.c and merge-sort-benchmark.c are the concrete starting point for the sorting algorithm benchmarks that you will write for this project. Both are standalone executables that can be compiled with the usual kind of Makefile entry. You can use the provided Makefile as a starting point.
  • The archive examples.tar contains example plots and plot scripts, as mentioned in the project assignment. This examples.zip contains the same in ZIP format.
