ITADS Frequently Asked Questions (FAQ)

How do I unlock a workstation in the UNIX room?

ctrl-alt-backspace-backspace

Press backspace twice in a row while holding down the Control and Alt keys. This will change to a different type of screen which reconnects your workstation to the server and presents a new login window.

How do I switch off that annoying beep?

Enter the following command in the terminal in order to disable the "bell": 

xset -b

How can I practice at home?

It is best to have Linux installed on your computer. There are many distributions, several are popular and easy to install. A good starting point may be Ubuntu, Fedora, or Knoppix.

Another possibility if you cannot install Linux is to run under Windows (or Mac OS X) using a virtual machine. The virtual appliance page provides an easy to use virtual machine image that you can download to get started quickly.

If you have a Mac, then it is easy to get set up for C development natively. You will need to install the Apple XCode Developer Tool. All other utilities that may be useful for the course are open source and can be obtained via macports.

How can I import exercises into Eclipse?

First of all, you need the C/C++ development tools plugin to be installed in Eclipse. The UNIX workstations in room B231c have this, but you need to launch a special version of Eclipse by giving the command: 

eclipse-cpp

If that crashes, then go to hh.se/helpdesk, log in with your student ID, and submit a new IT support case to tell them about your problem.

Step 1: import a project

  1. In the explorer windows right click select New->Project.
  2. Click C/C++->makefile Project with Existing Code and click Next.
  3. Click Browse and browse for the folder of your existing codes and click OK.
  4. Under Languages, uncheck the C++ option and under ToolChain for indexer settings select Linux GCC and click Finish.

Now if you already have a binary (executable) file in your project you can skip to Step 3: running the project.

Step 2: building the project

  1. Now you should have your project in the Project Explorer tab on the left hand side. Press Shift+F9 to open the Make Targets Window and click add and give it a name and press Ok.
    • Give your Target Binary File the same name as as your source file except without any extension (Linux/UNIX convention)
    • Now you should see your new target in the Make Targets window... but if you don't, then just close and re-open by Shift+F9 again (Eclipse bug).
  2. Select your target and press Build.
    • Note that if your project contains some errors you have to fix them then try to build again. You can use the Eclipse editor to find and fix the errors easily of course.

Step 3: running the project

  1. Now that you have built your project you can run it by right clicking on it in the Project Explorer and Selecting Run As->Local C/C++ Application.
  2. To give coomand-line arguments to your program before lunching it, you have to right-click on your project and select Run As->Run Configurations..., then in the Arguments tab you can enter your arguments and then press Run.
