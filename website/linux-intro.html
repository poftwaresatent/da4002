ITADS Introduction to Linux

C compilers exist for practically all operating systems. For the exercises and projects of this course, the GNU Compiler Collection (GCC) is used under Linux. This UNIX-like operating system gives you a large choice of editors and development environments, and frequently the best way to leverage the power of Linux is to use the command line. The separate tiny introduction to Linux (PDF) can help you to get started. Below, the most important aspects for working on ITADS exercises and projects are summarized.

Linux Basics

You interact with the Linux operating system by entering commands at a terminal. On the exercise web pages, commands are indicated with a special color scheme, like this: 

find . -type f

Do not confuse this with example code, which you have to enter into a text editor (such as the one integrated into Eclipse) and uses a different color scheme, like this: 

printf ("hello, world.\n");

The commands that you will be most likely to use frequently are:

mkdir
make a directory
example: 
mkdir -p itads/ex1
make a directory called ex1 inside the direcory itads
cd
change directory
example: 
cd itads/ex1
change into the directory ex1 which is underneath the directory itads
pwd
print working directory
example: 
pwd
will print something like "/home/username/itads/ex1" if you followed the above examples
ls
list directory contents
example: 
ls
will write a list of filenames and directory names
gcc
Run the C compiler on a .c source file.
example: 
gcc -Wall -o hello-world hello-world.c
Compile the source code contained in hello-world.c into an executable (in this case, the output will be stored in a file called hello-world)
Note the exercises are formulated for the Eclipse integrated development environment. It is thus unlikely that you will need to call the compiler from a command line, but it is still important to understand that the compiler is a program like any other on the system, and its operation can be precisely controlled from the command line.
gedit
Launch the gedit text editor
example: 
gedit hello-world.c &
Launch gedit and start editing the hello-world.c source file. If the file does not exist yet, gedit will create a new empty file for you.
Note: when launching gedit (or any other program that opens its own window), you should append "&" to the command. This makes the program start "in the background" and gives you back the command prompt. Otherwise, the terminal will not accept new commands (you can still type them, but they will not be executed).
nedit
Launch the nedit text editor
example: 
nedit hello-world.c &
Launch nedit and start editing the hello-world.c source file. Nedit may ask for confirmation to create the file.
Note: Nedit has some nice automatic indentation features that you may find very useful for editing source.
eclipse-cpp
Launch the Eclipse IDE (version with C/C++ developer tools installed on the machines in the Unix room B231c).

Writing, Compiling, and Running Programs Written in C

C source files are text files that have the extension .c and they are compiled by the C compiler (gcc) into executable machine code. In order to launch executables, you type the name of the executable into a terminal window, but you have to prefix the name with "./" (dot-slash) and you have to be in the directory where the file is stored. (This explanation is not entirely accurate, but suffices for the purposes of this course.) Note that you can also run executables directly from within the Eclipse IDE, as explained in the exercises.

If you do not have access to an IDE like Eclipse, you can use any text editor to write source code. Some choices, all of which should already be installed on the workstations used for the exercises, are:

gedit
a simple and easy to use editor
you launch it from a terminal window by typing "gedit &" (note the "&" which instructs the shell to launch gedit in the background: gedit will open its own separate window)
nedit
still simple, yet maybe a bit more appropriate than gedit for editing source code
you launch it from a terminal window by typing "nedit &" (note the "&" which instructs the shell to launch nedit in the background: nedit will open its own separate window)
emacs
an extensible and customizable text editor
you launch it from a terminal window by typing "emacs &" (again, note the "&")
vim
is a highly extensible modern version of the venerable vi editor
you launch it from a terminal window by typing "vim" (note the absence of "&" because vim launches within the terminal window)

Gedit is probably the easiest when you get started. Emacs and vim have a steeper learning curve but will vastly increase your efficiency in the long run. Another option is to use an integrated development environment (IDE) which lets you edit code, compile, run and debug it all within one application. Beside Eclipse, which is a widespread IDE and also used in this course, another popular alternative is KDevelop.
